# EtherCAT

Pure-Elixir EtherCAT master built on OTP. Runs on standard Ethernet hardware
with no RTOS, no kernel patch, and no proprietary NIC driver.

## How EtherCAT Works

Unlike traditional fieldbuses where the master polls each device in turn,
EtherCAT sends a single Ethernet frame (Ethertype `0x88A4`) around the entire
ring. Every slave reads and writes its data **in hardware as the frame passes
through** — no CPU interrupt, no software round-trip. By the time the frame
returns to the master, the full process image has been exchanged in one
network round-trip.

A few concepts worth knowing:

| Concept | What it does |
|---------|-------------|
| **Datagram** | The unit of work inside a frame: command + address + data + Working Counter (WKC). Multiple datagrams fit in one frame. |
| **FMMU** | Slave hardware that maps the master's flat *logical* address space directly onto each slave's *physical* register bank. |
| **SyncManager** | Guards the boundary between the EtherCAT side and the slave's local CPU — prevents both from touching the same buffer at once. |
| **ESM** | The slave state machine: INIT → PREOP → SAFEOP → OP. Inputs go live at SAFEOP; outputs only at OP. |
| **SII EEPROM** | Stores the slave's identity, PDO layout, and SyncManager config. Read by the master during INIT over I²C. |
| **CoE / SDO** | CANopen-over-EtherCAT mailbox protocol for reading and writing object dictionary entries. |
| **Distributed Clocks (DC)** | Hardware clock synchronization across every slave, accurate to under a microsecond. The first DC-capable slave becomes the network reference; all others discipline their clocks to it. Outputs a SYNC0 pulse that triggers cyclic tasks directly in slave silicon — independent of the master. |

## Why Elixir Without Hard Real-Time

The usual objection is that EtherCAT requires a real-time OS and Elixir cannot
provide one. That's true, but it misses where the hard timing constraint
actually lives.

Slaves do not execute at the moment the master's frame arrives. They execute
when the DC SYNC0 pulse fires — a hardware signal generated by the slave's own
clock, locked to the network reference via the distributed clock mechanism.
That pulse is accurate to sub-microseconds and is entirely independent of the
master's operating system.

The master's job is more relaxed: deliver a fresh frame somewhere inside each
DC cycle window (typically 1–4 ms). On a standard Linux system the BEAM
scheduler introduces jitter in the sub-millisecond range under light load, but
this can grow under contention. For cycle times of 1 ms or longer, a
well-configured system with the network process on an isolated CPU core stays
well inside the window. The cyclic domain loop aims to land the frame before
the midpoint of the window, giving headroom on both sides.

This won't suit every application — microsecond-level cycle times still need an
RTOS. But for the 1–4 ms range common in discrete I/O and motion pre-processing,
Elixir's concurrency model and fault-isolation are a better trade-off than
porting a C master to a language with cooperative scheduling.

## Quick Start

Requires `CAP_NET_RAW` or root on the target interface.

```bash
# Finite I/O test against physical slaves (no SIGTERM needed)
mix run --no-start examples/io_quick.exs --interface enp0s31f6

# Verbose mode — prints datagram hex and WKC
mix run --no-start examples/io_quick.exs --interface enp0s31f6 --verbose

# Dump SyncManager config after init
mix run --no-start examples/io_quick.exs --interface enp0s31f6 --dump-sm --cycles 1
```

Grant raw socket capability without running as root:

```bash
sudo setcap cap_net_raw+ep _build/dev/lib/ethercat/priv/raw_socket
```

## Usage

### Start the master

```elixir
EtherCAT.start(
  interface: "enp0s31f6",
  dc: %EtherCAT.DC.Config{cycle_ns: 1_000_000},
  slaves: [
    %EtherCAT.Slave.Config{name: :coupler},
    %EtherCAT.Slave.Config{name: :el1809, driver: MyApp.EL1809Driver},
    %EtherCAT.Slave.Config{name: :el2809, driver: MyApp.EL2809Driver}
  ],
  domains: [
    %EtherCAT.Domain.Config{id: :io, cycle_time_us: 1_000}
  ]
)

:ok = EtherCAT.await_running(5_000)
```

### Read inputs / write outputs

```elixir
# Subscribe to changes (sent as {:ethercat, :signal, :el1809, :input_0, value})
EtherCAT.subscribe(:el1809, :input_0, self())

# Synchronous read
{:ok, bit} = EtherCAT.read_input(:el1809, :input_0)

# Write outputs
:ok = EtherCAT.write_output(:el2809, :output_0, 1)
```

### Implement a slave driver

```elixir
defmodule MyApp.EL2809Driver do
  @behaviour EtherCAT.Slave.Driver

  @impl true
  def process_data_model(_config), do: %{output_0: 0x1600}

  @impl true
  def encode_signal(:output_0, _config, value), do: <<value::8>>

  @impl true
  def decode_signal(:output_0, _config, _raw), do: nil
end
```

### Raw frame access (advanced)

```elixir
alias EtherCAT.Bus.Transaction

bus = EtherCAT.bus()

{:ok, result} =
  EtherCAT.Bus.transaction(
    bus,
    Transaction.new()
    |> Transaction.fprd(station_addr, register, 2)    # read 2 bytes
    |> Transaction.fpwr(station_addr, register, data) # write data
  )
```

## Architecture

```
EtherCAT.Application
├── EtherCAT.Master              ← bus scan → config → running state machine
├── EtherCAT.SessionSupervisor
│   ├── EtherCAT.Bus             ← raw Ethernet (AF_PACKET) or UDP transport
│   ├── EtherCAT.DC              ← DC maintenance + lock/status monitoring
│   └── EtherCAT.Domain          ← cyclic LRW loop
├── EtherCAT.SlaveSupervisor
│   └── EtherCAT.Slave           ← per-slave ESM (INIT→PREOP→SAFEOP→OP)
│       ├── EtherCAT.Slave.SII   ← EEPROM identity + PDO map
│       └── EtherCAT.Slave.CoE   ← SDO mailbox
```

**Hot path:** `Domain` I/O is direct ETS reads/writes — no `GenServer` round-trips.
**Init path:** SII reads and slave configuration happen concurrently across all slaves.
**Redundancy:** Swap `transport: :raw` for `transport: :redundant` to add a backup interface with automatic failover.
